# PO Actual Receipt WMS/QR Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Cho phép mọi đợt giao PO (kể cả PO mua chủ động không có `requestId`) tạo WMS/QR, duyệt SL/CL theo số thực nhận có thể vượt PO, ghi nhận tồn kho/công nợ theo thực nhận và lưu tệp chứng minh với lazy-load.

**Architecture:** Giữ số lượng PO làm baseline audit, thêm `orderedQty`/variance vào item WMS và liên kết trực tiếp `purchase_order_delivery_batches.wms_transaction_id` cho PO chủ động. Tách nhánh tạo/chuẩn bị receipt theo nguồn `from_request` và `proactive`, dùng cùng RPC kiểm quyền để chấp nhận số thực nhận. Tệp chỉ lưu metadata trong transaction và object private trong Storage; signed URL chỉ sinh khi người dùng bấm xem/tải.

**Tech Stack:** React 18 + TypeScript, Supabase Postgres/RLS/Storage, `@supabase/supabase-js`, Vitest, SQL smoke tests.

## Global Constraints

- Cả `from_request` và `proactive_*` đều phải tạo được WMS/QR theo từng delivery batch.
- Duyệt SL/CL là thẩm quyền duy nhất chấp nhận chênh lệch; không tạo luồng duyệt PO bổ sung cho variance.
- PO giữ số đặt ban đầu; tồn kho và công nợ dùng `receivedQty` đã duyệt, tính theo đơn giá snapshot PO.
- Phiếu nhập/xuất tự sinh từ WMS; chỉ modal Duyệt SL/CL được chọn ảnh/tệp.
- Storage bucket `wms-transaction-attachments` là private; không lưu signed URL trong database và không dùng service key ở frontend.
- Tệp được lazy-load: mở chi tiết chỉ đọc metadata, bấm Xem/Tải mới gọi `createSignedUrl`.
- Backend/RPC là lớp quyết định cuối cùng cho quyền, quantity không âm, variance reason và idempotency.
- Không dùng `supabase db push` toàn bộ lịch sử vì cloud đang có migration remote-only; migration mới phải được apply đúng file và verify riêng.

```ts
export const validateReceiptQuantityLines = (
  transaction: Pick<Transaction, 'items'>,
  lines: ReceiptQuantityLine[],
): ReceiptQuantityLine[] => {
  const byIndex = new Map(lines.map(line => [line.index, line]));
  return transaction.items.map((item, index) => {
    const line = byIndex.get(index);
    if (!line || !Number.isFinite(line.quantity) || line.quantity < 0) {
      throw new Error(`Dòng ${index + 1}: Số lượng thực nhận không được âm.`);
    }
    const baseline = Number(item.quantity || 0);
    if (line.quantity !== baseline && !line.reason.trim()) {
      throw new Error(`Dòng ${index + 1}: Phải nhập lý do khi số lượng thực nhận lệch số lượng trên phiếu.`);
    }
    return { ...line, reason: line.reason.trim() };
  });
};

export const buildActualReceiptItems = (items: TransactionItem[], lines: ReceiptQuantityLine[]) =>
  items.map((item, index) => {
    const line = lines.find(candidate => candidate.index === index)!;
    const orderedQty = Number.isFinite(Number(item.orderedQty)) ? Number(item.orderedQty) : Number(item.quantity || 0);
    return { ...item, quantity: line.quantity, orderedQty, varianceReason: line.reason || item.varianceReason };
  });
```
